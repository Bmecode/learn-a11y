Exercise 1: Screen Reader Accessibility

In this exercise, you'll learn how to make your web content accessible to screen readers. First, let's get you set up with a screen reader based on your operating system.

Choosing a Screen Reader

macOS: VoiceOver

VoiceOver comes pre-installed on all Mac computers.

  1. Press Command + F5 to turn VoiceOver on/off, or
  2. Go to System Settings > Accessibility > VoiceOver
  3. Basic commands:
    • Navigate: Control + Option + arrow keys
    • Read page: Control + Option + A
    • Stop reading: Control

Windows: NVDA

NVDA is a free, open-source screen reader for Windows.

  1. Download NVDA from nvaccess.org
  2. Run the installer and follow the setup wizard
  3. Basic commands:
    • Start/Stop reading: Insert + Down Arrow
    • Navigate by headings: H
    • Navigate by links: Tab
    • Stop speaking: Control

Linux: Orca

Orca is the default screen reader for GNOME-based Linux distributions.

  1. Open Terminal and type: sudo apt-get install orca (Ubuntu/Debian)
  2. Launch Orca: Super + Alt + O
  3. Basic commands:
    • Start/Stop reading: Orca Modifier + H
    • Navigate: Arrow keys
    • Read current line: Orca Modifier + L

Browser Extension: ChromeVox

If you prefer not to install a system-wide screen reader, ChromeVox is a good alternative that works directly in Chrome.

  1. Install ChromeVox from the Chrome Web Store
  2. After installation, click the ChromeVox icon in your extensions bar to enable it
  3. Basic commands:
    • Navigate: Search + arrow keys
    • Click: Search + Space
    • Read from current position: Search + R

Practice Exercise

Once you have a screen reader set up, try the following:

  1. Navigate back to the workshop homepage using your screen reader
  2. Listen to how the exercise list is read
  3. Try to understand the page structure through headings

Notice how semantic HTML elements and proper heading structure make navigation easier for screen reader users.
